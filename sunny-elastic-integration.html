Integration

Cohere provides a family of production-grade large language models optimized for enterprise retrieval workloads — including embed models for dense vector search, rerank for late-stage relevance refinement, and Command-R for retrieval-augmented generation. All three are available natively from inside Elasticsearch via the Inference API, with no additional infrastructure to operate.

The integration ships in Elasticsearch 8.13+ and Elastic Cloud Serverless. Once configured, you can call Cohere models from ingest pipelines, search queries, or downstream retrieval stages using the same inference endpoint pattern you use for ELSER and other model providers.

What it does

The integration registers Cohere as an inference provider in your cluster. Once a model endpoint is created, Elasticsearch handles the API key, request batching, retry logic, and response parsing — so a Cohere embed call looks identical to any other inference call from the query DSL.

Three model families are supported out of the box: embed-english-v3.0 and embed-multilingual-v3.0 for dense embeddings (1024 dimensions), rerank-english-v3.0 and rerank-multilingual-v3.0 for cross-encoder reranking, and command-r / command-r-plus for grounded generation in RAG pipelines.

Setup

Create the inference endpoint via the PUT _inference API. You'll need a Cohere API key (free tier works for evaluation) and the model name. The endpoint becomes a reusable handle — reference it from your mapping for automatic embedding at ingest, or call it explicitly from a knn or text_similarity_reranker retriever.

Usage patterns

The most common pattern is a two-stage retrieval: first-pass BM25 or dense kNN to surface a candidate set, then Cohere rerank to reorder the top 100 or so by semantic relevance. This combination consistently outperforms either stage alone on standard IR benchmarks (BEIR, MIRACL), and the rerank call adds 50–120ms latency at typical batch sizes.

For RAG, embed your documents at ingest with embed-multilingual-v3.0, then pipe the retrieved passages into command-r-plus with citation tokens enabled — the returned response includes spans that link back to specific source documents, which makes auditing the model's grounding straightforward.

Cost and scaling

Cohere bills per million input tokens, separate prices for embed, rerank, and generate. For high-volume ingest, batch your embed calls (the integration handles batching automatically up to the API limit of 96 documents per request). Rerank calls are billed per search and scale linearly with the candidate set size — tune your first-stage k accordingly.

For latency-sensitive workloads, deploy in the same cloud region as your Elasticsearch cluster (Cohere offers AWS, Azure, and GCP regional endpoints) to keep round-trip times under 100ms.
